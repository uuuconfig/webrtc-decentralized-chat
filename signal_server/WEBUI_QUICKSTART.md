# 信令服务器 WebUI - 快速启动指南

## 📦 安装依赖

```bash
# 安装信令服务器依赖
pip install -r signal_server/requirements.txt
```

## 🚀 启动服务器

```bash
# 启动信令服务器（包含 WebUI）
python signal_server/server.py
```

启动后会看到类似的日志输出：
```
2026-04-07 14:00:00 [INFO] WebUI 访问地址: http://localhost:5011/dashboard
2026-04-07 14:00:00 [INFO] 信令服务器启动在 ws://0.0.0.0:5010
```

## 🌐 访问 WebUI

打开浏览器访问：**http://localhost:5011/dashboard**

## 📊 WebUI 功能说明

### 1. 服务器状态面板
- **启动时间**：服务器启动的具体时间
- **运行时长**：自启动以来的运行时间（自动更新）
- **监听地址**：信令服务器的 WebSocket 地址
- **WebUI 地址**：仪表板的访问地址

### 2. 实时统计面板
显示以下实时指标（每秒更新）：
- **在线用户**：当前连接的用户数
- **活跃群聊**：当前存在的群聊数
- **待处理连接**：正在进行的 P2P 连接请求数
- **消息吞吐**：每秒处理的消息数
- **总消息数**：自启动以来处理的总消息数
- **连接成功率**：成功建立的连接占比
- **平均连接时间**：建立连接的平均耗时

### 3. 在线用户列表
显示所有在线用户的信息：
- 用户名
- 连接时间（具体时刻）
- 连接时长（已连接多久）
- 在线状态指示器

用户上线/下线时实时更新。

### 4. 群聊管理面板
显示所有群聊的信息：
- 群 ID（前 12 个字符）
- 创建者
- 成员数（总数/在线数）
- 创建时间
- 群成员列表（可展开查看）

群创建/删除/成员变化时实时更新。

### 5. 待处理连接面板
显示正在进行的 P2P 连接请求：
- 发起者 → 接收者
- 连接 ID（前 16 个字符）
- 发起时间
- 已耗时（秒）

连接请求发起/完成时实时更新。

### 6. 实时日志面板
显示服务器的实时日志：
- 时间戳（精确到秒）
- 日志级别（INFO/WARNING/ERROR）
- 日志类别（注册/连接/错误/群聊等）
- 日志消息

**功能：**
- **日志筛选**：按类别筛选（全部/注册/连接/错误/群聊）
- **日志搜索**：按用户名或关键词搜索
- **自动滚动**：新日志自动追加到底部
- **历史日志**：保留最近 100 条日志

### 7. 主题切换
点击右上角的主题按钮可切换浅色/深色主题，选择会自动保存。

## 🧪 测试步骤

### 测试 1：基本连接
1. 启动服务器
2. 打开 WebUI（http://localhost:5011/dashboard）
3. 验证连接状态显示"已连接"
4. 验证服务器状态面板显示正确的启动时间和地址

### 测试 2：用户上线/下线
1. 启动一个客户端：`cd client && python app.py --port 8000`
2. 在客户端输入用户名并加入聊天
3. 在 WebUI 中验证：
   - 在线用户数增加
   - 用户列表中出现新用户
   - 日志中显示"用户注册"消息
4. 关闭客户端
5. 在 WebUI 中验证：
   - 在线用户数减少
   - 用户从列表中移除
   - 日志中显示"用户离线"消息

### 测试 3：群聊操作
1. 启动两个客户端
2. 在第一个客户端创建群聊
3. 在 WebUI 中验证：
   - 活跃群聊数增加
   - 群聊列表中出现新群
   - 日志中显示"创建群链接"消息
4. 在第二个客户端加入群聊
5. 在 WebUI 中验证：
   - 群聊的在线成员数增加
   - 群成员列表更新
   - 日志中显示"加入群"消息

### 测试 4：P2P 连接
1. 启动两个客户端
2. 在第一个客户端向第二个客户端发起连接
3. 在 WebUI 中验证：
   - 待处理连接数增加
   - 待处理连接列表中显示连接信息
   - 日志中显示"转发 offer"消息
4. 连接建立后验证：
   - 待处理连接数减少
   - 日志中显示"转发 answer"消息
   - 连接成功率和平均连接时间更新

### 测试 5：日志筛选和搜索
1. 执行上述测试操作产生各种日志
2. 点击不同的筛选按钮（注册/连接/错误/群聊）
3. 验证日志列表只显示对应类别的日志
4. 在搜索框输入用户名或关键词
5. 验证日志列表只显示匹配的日志

### 测试 6：主题切换
1. 点击右上角的主题按钮
2. 验证页面从浅色切换到深色（或反之）
3. 刷新页面
4. 验证主题设置被保存

### 测试 7：自动重连
1. 启动服务器和 WebUI
2. 在浏览器开发者工具中模拟网络断开
3. 验证连接状态显示"已断开"
4. 恢复网络连接
5. 验证 WebUI 自动重新连接
6. 验证连接状态显示"已连接"

## 🔧 故障排除

### WebUI 无法连接
- 检查服务器是否正常运行
- 检查浏览器控制台是否有错误信息
- 尝试刷新页面
- 检查防火墙设置

### 数据不更新
- 检查 WebSocket 连接状态
- 查看浏览器控制台是否有错误
- 尝试重新启动服务器

### 日志不显示
- 检查服务器是否有日志输出
- 尝试执行测试操作产生新日志
- 检查日志筛选设置
