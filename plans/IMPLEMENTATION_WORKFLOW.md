# 本地 JSON 存储实现工作流程

## 🔄 完整工作流程图

### 1. 用户登录流程

```
┌─────────────────────────────────────────────────────────────┐
│ 用户在浏览器输入用户名，点击登录                             │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端发送 WebSocket 消息: { type: "register", username }     │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 后端 app.py 接收 register 消息                              │
│ - 调用 storage_manager.init_user_folder(username)          │
│ - 创建 data/{username}/ 文件夹（如不存在）                 │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 后端返回 registered 消息给前端                              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端接收 registered 消息                                    │
│ - 调用 loadFromBackend(username)                           │
│ - 发送 GET /api/data/{username}/all                        │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 后端处理 GET /api/data/{username}/all                      │
│ - 调用 storage_manager.load_all_data(username)             │
│ - 读取所有 JSON 文件                                        │
│ - 返回完整数据                                              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端接收数据                                                │
│ - 加载到 localStorage                                       │
│ - 初始化 UI（聊天列表、联系人、设置等）                    │
│ - 启动自动同步机制 startAutoSync()                         │
└─────────────────────────────────────────────────────────────┘
```

### 2. 发送消息流程（关键操作 - 实时保存）

```
┌─────────────────────────────────────────────────────────────┐
│ 用户输入消息，点击发送                                      │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端 onSendMessage()                                        │
│ - 创建消息对象                                              │
│ - 添加到 this.chatHistory[target]                          │
│ - 保存到 localStorage                                       │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端立即调用 syncToBackend('chat_history', data)           │
│ - 发送 POST /api/data/{username}/chat_history              │
│ - 请求体: { data: this.chatHistory }                       │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 后端处理 POST /api/data/{username}/chat_history            │
│ - 验证用户名                                                │
│ - 调用 storage_manager.save_data(username, 'chat_history') │
│ - 写入 data/{username}/chat_history.json                   │
│ - 返回 { success: true }                                    │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端接收响应                                                │
│ - 如果成功，继续                                            │
│ - 如果失败，重试（最多 3 次）                              │
└─────────────────────────────────────────────────────────────┘
```

### 3. 修改设置流程（定时保存）

```
┌─────────────────────────────────────────────────────────────┐
│ 用户修改设置（如主题、通知等）                              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端 changeTheme() / toggleNotification() 等                │
│ - 更新 this.settings 对象                                   │
│ - 保存到 localStorage                                       │
│ - 调用 markDirty('settings')                               │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端定时器（每 10 秒）                                      │
│ - 检查脏数据列表                                            │
│ - 如果 'settings' 被标记为脏                               │
│ - 调用 syncToBackend('settings', this.settings)            │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 后端处理 POST /api/data/{username}/settings                │
│ - 写入 data/{username}/settings.json                       │
│ - 返回成功                                                  │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端清除脏标记                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4. 添加联系人流程（关键操作 - 实时保存）

```
┌─────────────────────────────────────────────────────────────┐
│ 用户输入联系人名称，点击添加                                │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端 addContact()                                           │
│ - 验证联系人是否存在                                        │

└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 前端立即调用 syncToBackend('contacts', this.contacts)      │
│ - 发送 POST /api/data/{username}/contacts                  │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│ 后端处理并写入 data/{username}/contacts.json               │
└─────────────────────────────────────────────────────────────┘
```

### 5. 登出流程

```
┌─────────────────────────────────────────────────────────────┐
```

## 📊 数据同步时序图

```
时间轴 ────────────────────────────────────────────────────────

T0: 用户登录
    ├─ 前端: 发送 register 消息
    ├─ 后端: 初始化用户文件夹
    ├─ 前端: 请求 GET /api/data/{username}/all
    ├─ 后端: 返回所有数据
    └─ 前端: 加载到 localStorage，启动自动同步

T1: 用户发送消息
    ├─ 前端: 更新 chatHistory，保存到 localStorage
    ├─ 前端: 立即调用 syncToBackend('chat_history')
    ├─ 后端: 写入 chat_history.json
    └─ 后端: 返回成功

T2: 用户修改设置
    ├─ 前端: 更新 settings，保存到 localStorage
    ├─ 前端: 标记 'settings' 为脏
    └─ 等待定时器...

T3: 定时器触发（10 秒后）
    ├─ 前端: 检查脏数据
    ├─ 前端: 发现 'settings' 脏，调用 syncToBackend('settings')
    ├─ 后端: 写入 settings.json
    ├─ 后端: 返回成功
    └─ 前端: 清除脏标记

T4: 用户添加联系人
    ├─ 前端: 更新 contacts，保存到 localStorage
    ├─ 前端: 立即调用 syncToBackend('contacts')
    ├─ 后端: 写入 contacts.json
    └─ 后端: 返回成功

T5: 用户登出
    ├─ 前端: 停止自动同步
    ├─ 前端: 清空 localStorage
    └─ 前端: 返回登录界面
```

## 🛡️ 错误处理流程

### 网络错误处理

```
前端发送 POST /api/data/{username}/chat_history
    │
    ├─ 成功 (200) ──→ 继续
    │
    ├─ 失败 (5xx) ──→ 重试 (最多 3 次)
    │                  │
    │                  ├─ 重试成功 ──→ 继续
    │                  │
    │                  └─ 重试失败 ──→ 记录日志，保留 localStorage
    │
    └─ 网络错误 ──→ 重试 (最多 3 次)
                     │
                     ├─ 重试成功 ──→ 继续
                     │
                     └─ 重试失败 ──→ 记录日志，保留 localStorage
```

### 文件系统错误处理

```
后端尝试写入 JSON 文件
    │
    ├─ 成功 ──→ 返回 { success: true }
    │
    ├─ 文件不存在 ──→ 创建新文件 ──→ 写入 ──→ 返回成功
    │
    ├─ 权限错误 ──→ 返回 500 错误 ──→ 前端重试
    │
    └─ JSON 解析错误 ──→ 返回 400 错误 ──→ 前端记录日志
```

## 📈 性能考虑

### 缓冲机制
- 不是每条消息都立即写入文件
- 使用内存缓冲，定期批量写入
- 减少磁盘 I/O 次数

### 并发处理
- 使用文件锁防止并发写入冲突
- 或使用队列机制序列化写入操作

### 大数据处理
- 对于大量消息，考虑分页加载
- 只在内存中保留最近的消息
- 旧消息存档到单独的文件
