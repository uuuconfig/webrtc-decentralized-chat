# P2P Chat 改进计划总结

## 📋 项目概述

本计划针对P2P Chat应用的6个主要问题进行系统改进，涉及前端UI/UX、后端逻辑和信令服务器三个层面。

---

## 🎯 问题清单与解决方案

### 问题1：UI布局问题 - 文件传输和群成员列表挡住输入框

**优先级**：🔴 高  
**影响范围**：用户体验  
**复杂度**：中等

**问题描述**：
- 文件传输面板覆盖输入框
- 群成员面板覆盖聊天区域
- 小屏幕上输入框不可见

**解决方案**：
- 重新设计CSS布局，使用flexbox
- 输入框使用sticky定位，始终可见
- 文件传输面板改为侧边栏
- 群成员面板改为右侧栏
- 移动设备上使用模态框

**涉及文件**：
- `client/static/style.css` - CSS布局修改
- `client/static/index.html` - HTML结构调整

**预期效果**：
✅ 输入框在所有屏幕尺寸上都可见  
✅ 文件传输和群成员面板不遮挡输入框  
✅ 移动设备上有良好的用户体验

### 问题2：群聊消息缺少发送者信息和IP地址

**优先级**：🟡 中  
**影响范围**：功能完整性  
**复杂度**：高

**问题描述**：
- 群聊消息只显示内容，看不到谁发的
- 没有显示发送者的IP地址
- 群成员列表中也没有IP信息

**解决方案**：

1. **后端修改**：
   - 在P2P连接建立时获取对端IP地址
   - 维护用户IP映射表
   - 在群消息中附加发送者IP信息

2. **前端修改**：
   - 修改消息渲染，添加发送者头部
   - 显示格式：`[发送者名字] (IP地址) 时间`
   - 在群成员列表中显示IP地址

**涉及文件**：
- `client/p2p.py` - 获取IP地址
- `client/app.py` - 维护IP映射表
- `client/static/app.js` - 前端显示逻辑
- `client/static/index.html` - HTML结构

**数据结构扩展**：
```javascript
// 扩展connectionStates
connectionStates: {
  'username': {
    state: 'connected',
    ip: '192.168.1.100'
  }
}

// 群消息格式
{
  type: 'group_p2p_message',
  groupId: 'xxx',
  from: 'username',
  fromIp: '192.168.1.100',
  content: 'message',
  timestamp: 'iso-string'
}
```

**预期效果**：
✅ 群聊消息显示发送者名字  
✅ 群聊消息显示发送者IP地址  
✅ 群成员列表显示每个成员的IP  
✅ 用户能清楚地识别谁在说话

### 问题3：群聊选中状态不更新

**优先级**：🔴 高  
**影响范围**：用户体验  
**复杂度**：低

**问题描述**：
- 选中群A后显示深色
- 切换到群B，群A仍然显示深色
- 选中状态标记不正确

**根本原因**：
- `selectChat()`方法更新了`currentChat`但没有更新DOM
- `renderGroupList()`每次重新生成列表，但旧的`active`类没有被移除

**解决方案**：
1. 在`selectChat()`中先移除所有群列表项的`active`类
2. 然后添加新选中群的`active`类
3. 确保`renderGroupList()`中只有当前选中的群有`active`类

**涉及文件**：
- `client/static/app.js` - `selectChat()`和`renderGroupList()`方法

**预期效果**：
✅ 选中状态正确显示  
✅ 切换群时深色标记正确更新  
✅ 用户体验流畅

---

### 问题4：增加客户端删除群聊功能

**优先级**：🟡 中  
**影响范围**：功能完整性  
**复杂度**：中等

**问题描述**：
- 用户无法删除已加入的群聊
- 需要手动清理本地数据

**解决方案**：

1. **前端UI**：
   - 在群成员面板顶部添加"删除群"按钮
   - 显示确认对话框

2. **前端逻辑**：
   - 添加`deleteGroup()`方法
   - 从本地存储中删除群信息
   - 重新渲染群列表

3. **后端逻辑**：
   - 添加`leave_group_link`消息处理
   - 清理本地群数据
   - 通知信令服务器

4. **信令服务器**：
   - 处理用户离开群消息
   - 通知其他群成员

**涉及文件**：
- `client/static/index.html` - 添加删除按钮
- `client/static/app.js` - 删除逻辑
- `client/app.py` - 后端处理
- `signal_server/server.py` - 信令服务器处理

**预期效果**：
✅ 用户可以删除群聊  
✅ 删除后群从列表中消失  
✅ 群历史记录被清除  
✅ 其他群成员收到离开通知

### 问题5：刷新网页后群成员显示未连接

**优先级**：🟢 低  
**影响范围**：稳定性  
**复杂度**：中等

**问题描述**：
- 用户刷新浏览器后，群成员列表显示"未连接"
- 但实际上P2P连接是正常的，能收到消息
- 连接状态显示不准确

**根本原因**：
- 刷新后浏览器WebSocket重新连接
- 前端状态被清空，`connectionStates`被重置
- 后端没有在浏览器重新连接时恢复连接状态

**解决方案**：

1. **后端修改**：
   - 浏览器WebSocket重新连接时，发送当前所有已建立的P2P连接状态
   - 发送群成员的连接状态

2. **前端修改**：
   - 接收并恢复连接状态
   - 更新`connectionStates`对象
   - 重新渲染用户列表和群成员面板

**涉及文件**：
- `client/app.py` - 恢复连接状态
- `client/static/app.js` - 接收并应用连接状态

**消息格式**：
```javascript
{
  type: 'restore_connection_states',
  states: {
    'username1': { state: 'connected', ip: '192.168.1.100' },
    'username2': { state: 'disconnected', ip: null }
  }
}
```

**预期效果**：
✅ 刷新后连接状态正确显示  
✅ 群成员列表显示准确的连接状态  
✅ 用户体验一致

---

### 问题6：客户端重启后自动重连群聊

**优先级**：🔴 高  
**影响范围**：用户体验  
**复杂度**：中等

**问题描述**：
- 客户端重启后，需要重新输入群链接ID才能重新连接
- 用户希望点击群聊时自动重连

**根本原因**：
- 群信息保存在浏览器localStorage中
- 后端Python服务重启后，群数据被清空
- 前端没有在选择群时自动向后端发送加入请求

**解决方案**：

1. **前端修改**：
   - 修改`selectChat()`方法
   - 如果是群聊且没有连接到该群，自动发送加入请求
   - 添加标志位防止重复加入

2. **后端修改**：
   - 修改`join_group_link`处理逻辑
   - 检查是否已经加入过
   - 如果已加入，直接返回已有成员列表

**涉及文件**：
- `client/static/app.js` - `selectChat()`方法
- `client/app.py` - 群加入逻辑

**前端状态跟踪**：
```javascript
// 添加群加入状态跟踪
groupJoinedStatus: {
  'groupId': true  // 已加入
}
```

**预期效果**：
✅ 客户端重启后自动重连群聊  
✅ 点击群聊时自动加入  
✅ 不需要重新输入群链接ID  
✅ 能正常收发消息

## 🔧 技术架构变更

### 前端数据结构

```javascript
// 原有结构
connectionStates: {
  'username': 'connected'  // 字符串
}

// 新结构
connectionStates: {
  'username': {
    state: 'connected',
    ip: '192.168.1.100',
    lastUpdate: timestamp
  }
}

// 新增：群加入状态跟踪
groupJoinedStatus: {
  'groupId': true
}
```

### 后端数据结构

```python
# 新增：用户IP映射表
state = {
  'user_ips': {
    'username': '192.168.1.100'
  },
  'group_peers': {},  # 现有
  'my_groups': {},    # 现有
}
```

### 消息格式扩展

```javascript
// 群消息格式（新增fromIp字段）
{
  type: 'group_p2p_message',
  groupId: 'xxx',
  from: 'username',
  fromIp: '192.168.1.100',  // 新增
  content: 'message',
  timestamp: 'iso-string'
}

// 连接状态恢复消息（新增）
{
  type: 'restore_connection_states',
  states: {
    'username': {
      state: 'connected',
      ip: '192.168.1.100'
    }
  }
}
```
