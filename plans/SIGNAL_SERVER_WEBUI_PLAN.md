# 信令服务器 WebUI 实现计划

## 📋 项目概述

为 P2P Chat 信令服务器创建一个本地 WebUI 仪表板，用于实时监控服务器状态、用户连接、群聊管理和性能指标。

## 🎯 核心功能

### 1. 服务器状态监控
- **启动时间**：服务器启动的时间戳
- **运行时长**：格式化的运行时间（如 "2天 3小时 45分钟"）
- **监听地址**：ws://0.0.0.0:5010
- **WebUI 访问地址**：http://localhost:5010/dashboard

### 2. 在线用户统计
- **当前在线用户数**：实时计数
- **用户列表**：
  - 用户名
  - 连接时间
  - 连接时长
  - 状态指示器（在线/离线）
- **用户上线/下线历史**：最近 50 条记录

### 3. 群聊管理
- **活跃群聊数量**：当前存在的群数
- **群聊列表**：
  - 群 ID
  - 创建者
  - 总成员数
  - 在线成员数
  - 创建时间
  - 群成员详情（可展开）

### 4. 连接状态监控
- **待处理 Offer 数量**：当前正在进行的连接请求数
- **连接列表**：
  - 发起者 → 接收者
  - 连接 ID
  - 发起时间
  - 状态（pending/completed）

### 5. 服务器性能指标
- **消息吞吐量**：每秒处理的消息数（实时更新）
- **总消息数**：自启动以来处理的总消息数
- **连接成功率**：成功建立的连接 / 总连接请求
- **平均连接时间**：建立连接的平均耗时

### 6. 实时日志查看
- **日志流**：最近 100 条日志记录
- **日志类型筛选**：
  - 全部
  - 用户注册
  - 连接信息
  - 错误信息
  - 群聊操作
- **日志搜索**：按用户名或关键词搜索
- **自动滚动**：新日志自动追加到底部

### 前端实现

**WebSocket 连接管理**：
```javascript
const ws = new WebSocket('ws://localhost:5010/dashboard');
ws.onmessage = (event) => {
    const msg = JSON.parse(event.data);
    if (msg.type === 'stats_update') {
        updateStats(msg.data);
    } else if (msg.type === 'log') {
        addLog(msg.data);
    }
};
```

**数据更新策略**：
- 统计数据：每秒推送一次
- 日志：实时推送（新日志立即发送）
- 用户列表：用户上线/下线时推送
- 群聊列表：群创建/删除/成员变化时推送

## 📊 性能考虑

- **日志缓冲**：服务器端维护最近 100 条日志，新 WebUI 连接时发送历史日志
- **数据节流**：统计数据每秒推送一次，避免过频繁的更新
- **内存管理**：使用 deque 限制日志和消息时间戳的数量
- **连接管理**：WebUI 连接断开时自动清理
